# 💰 Module 17 — Cost Optimization for GenAI

> **Preventing Bill Shock** — Strategies to run AI at scale without bankrupting the company.

---

## 🧠 1️⃣ Intuition — The Economics of LLMs

In traditional cloud computing, compute (EC2) and storage (S3) drive costs.
In GenAI, **Tokens** drive costs.

Every word sent to the model (Input) and every word generated by the model (Output) costs money.
- **Input Tokens** are relatively cheap.
- **Output Tokens** are usually 3x to 5x more expensive than input tokens.

**The Multiplier Effect**: In a RAG application, a user might send a 10-word query. But your system retrieves 5 documents (2,000 words), appends a system prompt (500 words), and sends it all to the model. A 10-word query just cost you 2,510 input tokens.

## ⚙️ 2️⃣ Internal Working — AWS Pricing Models

### Amazon Bedrock Pricing

Bedrock offers two primary pricing models:

1. **On-Demand (Pay-as-you-go)**
   - No upfront commitment.
   - Billed per 1,000 input and output tokens.
   - Best for: Development, unpredictable workloads, spiky traffic.

2. **Provisioned Throughput**
   - You reserve dedicated compute capacity (Model Units).
   - Billed hourly, regardless of how many tokens you process.
   - Minimum commitment: 1 month (or 6 months for a discount).
   - Best for: High-volume production, latency-sensitive apps, custom fine-tuned models.

### Break-Even Analysis

When do you switch from On-Demand to Provisioned Throughput?
You must calculate the break-even point.

*Example (Hypothetical pricing for Claude 3 Haiku)*:
- On-Demand: $0.25 / 1M Input tokens, $1.25 / 1M Output tokens.
- Provisioned: $10.00 / hour.

If your token usage costs > $10.00 per hour on average, you should switch to Provisioned Throughput.

### Auxiliary Costs (The Hidden Drivers)
- **OpenSearch Serverless**: Billed per OCU. Minimum 4 OCUs (2 ingest, 2 search) = ~$700/month.
- **Data Transfer**: Moving massive datasets into S3 for Knowledge Bases.

## 🏗️ 3️⃣ Production Usage — Optimization Strategies

### 1. Semantic Caching

If 100 users ask "What is the holiday policy?", don't pay Bedrock 100 times.
Cache the response.

**Implementation**:
1. User asks question.
2. Embed the question (cheap).
3. Search a cache (e.g., DynamoDB or Redis) for similar embeddings using cosine similarity.
4. If similarity > 0.95, return cached answer. Else, call Bedrock and save to cache.

### 2. Model Routing (Cascading)

Not every query needs Claude 3.5 Sonnet.

**Implementation**:
Use a fast, cheap model (Claude 3 Haiku or Titan) as a router.
- If the query is simple ("reset password"): Router handles it.
- If the query is complex ("analyze this Python code"): Router passes it to Sonnet.

### 3. Context Window Optimization

Don't send the entire 50-page PDF in the prompt.
- **Strict Chunking**: Keep chunks small (300 tokens) in RAG.
- **Top-K Reduction**: Only send the Top 3 relevant chunks to the model, not Top 10.

### 4. Prompt Caching (Anthropic Specific)

Claude 3.5 supports prompt caching. If you send the same massive system prompt or context document frequently within a 5-minute window, you get a significant discount (up to 90%) and faster latency on those cached tokens.

## 🎮 4️⃣ GameDay Relevance

### The Cost Challenge

GameDay often features a "Cost Optimization" quest. You will be given a working, but hugely expensive architecture.

**How to win it:**
1. Check Bedrock models. Is the team using Claude Opus for a simple classification task? Downgrade to Haiku or Titan.
2. Check OpenSearch Serverless. Is it scaled to 20 OCUs? Reduce the max capacity.
3. Check RAG config. Is the `numberOfResults` set to 50? Reduce it to 3.

## 💼 5️⃣ Interview Perspective

### Q: "Your GenAI application is a massive success, but the AWS bill is growing exponentially. Walk me through how you would optimize costs."

**Model Answer**:
> "I would attack this in three phases:
> 
> 1. **Measurement & Visibility**: First, I'd implement cost allocation tags on the Bedrock API calls and set up AWS Cost Anomaly Detection to pinpoint exactly which features are driving token usage.
> 
> 2. **Architectural Optimization**: I'd implement Semantic Caching so we don't pay for repeated queries. Next, I'd implement Model Routing—defaulting to smaller, cheaper models like Claude 3 Haiku for basic tasks, and only escalating to Sonnet for complex reasoning.
> 
> 3. **Commitment Discounts**: Once our baseline traffic stabilizes, I would run a break-even analysis and move the predictable portion of our traffic to Bedrock Provisioned Throughput with a 6-month commitment to significantly reduce the per-token cost."
