Domain 1: Agentic Architecture & Orchestration

Design multi-agent systems, manage task decomposition, and implement hub-and-spoke models.

Hub-and-Spoke Pattern

The recommended architecture for complex tasks. A Supervisor (Router) agent breaks down a prompt, delegates to specialized sub-agents, and synthesizes the final output.

  • Supervisor: Responsible for planning, delegation, and final review.
  • Sub-agents: Highly specialized prompts (e.g., "You are an expert at searching logs").

When to use Multi-Agent?

Avoid multi-agent overhead for simple workflows. Use them when:

  • The task requires multiple distinct domains of expertise.
  • Context windows would be overwhelmed by combining all tools into one agent.
  • You need distinct validation loops.

Domain 2: Claude Code Configuration

Mastering CLAUDE.md hierarchies and CI/CD pipelines.

CLAUDE.md Hierarchy

Claude Code uses a cascading configuration system. Rules in a sub-directory's CLAUDE.md will append to or override rules from the parent directory.

  • Project Level: Global styling, architecture guidelines.
  • Directory Level: Specific test commands, linting rules.

CI/CD Integration (-p flag)

To run Claude Code non-interactively in CI pipelines (e.g., GitHub Actions for PR reviews), use the -p (print) flag. Without it, the pipeline will hang waiting for user input.

Domain 3: Prompt Engineering & Structured Output

Programmatic Enforcement

Never rely on "Please output JSON" in your system prompt for production pipelines. The LLM might include conversational filler like "Here is your JSON:" which breaks JSON.parse().

Tool Use for Schemas

Provide a single Tool (e.g., record_data) and force Claude to use it (via tool_choice). Claude's internal training heavily prioritizes matching Tool schemas perfectly, guaranteeing a strictly typed JSON object back to your application.

Domain 4: Tool Design & MCP

The Model Context Protocol

MCP standardizes how AI connects to external data. It establishes a secure boundary between the Host (Claude Desktop/Claude Code) and external environments.

  • Tools: Actions the LLM can take (e.g., write_file).
  • Resources: Data the LLM can read (e.g., postgres://table).
  • Prompts: Reusable prompt templates exposed by the server.

Security Boundaries

The LLM host never executes code. It requests an MCP server to execute a Tool. The MCP Server executes the logic locally and returns the result to the Host. The Host user must explicitly approve sensitive tool calls.

Domain 5: Context Management & Reliability

Prompt Caching Thresholds

Prompt Caching drastically reduces costs (up to 90%) and latency for long contexts. However, caching only activates if the prefix is at least 1024 tokens.

Context Structuring

To maximize cache hits, place static content (System prompts, long reference PDFs, massive few-shot examples) at the beginning of your context window. Place dynamic, changing content (User queries, recent chat history) at the end.

What is the minimum token threshold to activate Prompt Caching?

1024 tokens. Any static prefix shorter than this will not be cached, even if explicitly marked with `cache_control`.

How do you run Claude Code non-interactively in a CI/CD pipeline?

Use the -p (print) flag. Example: claude -p "Review this PR". This prevents the CLI from hanging while waiting for user input.

In a multi-agent system, what is the role of the Supervisor (Router) Agent?

The Supervisor is responsible for task decomposition, delegating to sub-agents, evaluating results, and synthesizing the final output.

How should you reliably guarantee a strict JSON output?

Use Tool Use (Function Calling) and enforce the tool_choice. Do not rely solely on system prompt instructions like "Output only JSON".
