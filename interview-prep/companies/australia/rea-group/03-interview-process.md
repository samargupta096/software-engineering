## ðŸ”„ Interview Stages

### Stage 1: Initial Phone Screen (20-40 minutes)

**Who:** Recruiter or Hiring Manager

**Focus:**
- Career history and aspirations
- Understanding of REA Group
- Role expectations
- Salary expectations
- Availability

**Preparation:**

---

### Stage 2: Technical Interview(s)

**Who:** 2+ interviewers from REA Group (Tech Leads, Engineers)

**Format:** 60-90 minutes

**Components:**

1. **Experience Deep Dive (30-40 min)**
   - Recent projects and contributions
   - Solutions to client/customer problems
   - Research, design, development involvement
   - Use **STAR method** for answers

2. **Coding/Pairing Interview (30-40 min)**
   - Live coding session
   - Observe problem-solving approach
   - Team collaboration style
   - Focus on thought process, not just solution

3. **Technical Domain Discussion (20-30 min)**
   - Domain knowledge (PropTech, cloud, AI/ML)
   - Architecture discussions
   - Technology choices and trade-offs

**Topics to Prepare:**

| Area | Topics |
|------|--------|
| **Languages** | Ruby, Scala, Java, Python, JavaScript |
| **Cloud** | AWS services, multi-region architecture |
| **Architecture** | Microservices, API design, event-driven |
| **Data** | BigQuery, data pipelines, analytics |
| **AI/ML** | Recommendation systems, image processing |
| **PropTech** | Property search, valuation, user experience |

## ðŸŽ¯ STAR Method Framework

REA Group emphasizes the **STAR method** for answering questions:

**S**ituation - Set the context  
**T**ask - Describe your responsibility  
**A**ction - Explain what you did  
**R**esult - Share the outcome

### Example Question:
> "Tell me about a time you solved a complex technical problem"

**Answer:**
- **S:** Our property search was slow with 5M+ listings
- **T:** Tasked to improve search performance by 50%
- **A:** Implemented Elasticsearch with geospatial indexing, added caching layer, optimized queries
- **R:** Reduced search latency from 2s to 400ms (80% improvement), increased user engagement by 25%

### Problem Types

1. **Data Structures & Algorithms**
   - Arrays, strings, hash maps
   - Trees, graphs
   - Sorting, searching
   - Dynamic programming

2. **System Design** (for senior roles)
   - Property search system
   - Recommendation engine
   - Image processing pipeline
   - Real-time analytics

3. **Domain-Specific**
   - Geospatial queries
   - Property matching algorithms
   - Price prediction models

### Sample Problems

**Problem 1: Property Search**
```
Design a system to search properties by:
- Location (radius-based)
- Price range
- Property type
- Bedrooms/bathrooms
Handle 10M listings, 100K concurrent users
```

**Problem 2: Recommendation System**
```
Given user browsing history and property features,
recommend 10 most relevant properties.
Consider: location preference, price range, property type
```

**Problem 3: Image Compliance**
```
Design a system to detect and flag images containing:
- Watermarks/logos
- Contact information (phone, email)
Process 100K images/day with <1s latency
```
