### CP Systems (Consistency + Partition Tolerance)

When a partition occurs, the system blocks until consistency is ensured.

| Database | Use Case |
|----------|----------|
| **MongoDB** | When configured for strong consistency |
| **Redis** (single master) | Atomic operations |
| **Zookeeper** | Configuration management |
| **HBase** | When strong consistency needed |
| **etcd** | Distributed configuration |

### AP Systems (Availability + Partition Tolerance)

System remains available but may return stale data.

| Database | Use Case |
|----------|----------|
| **Cassandra** | High availability, eventual consistency |
| **DynamoDB** | Global applications |
| **CouchDB** | Offline-first apps |
| **Riak** | High availability needs |
