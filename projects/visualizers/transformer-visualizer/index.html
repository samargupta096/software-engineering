01

Input Embeddings & Positional Encoding

Transformers don't read text. They read vectors. First, words are mapped to semantic embeddings. But because Transformers process all words in parallel, they have no sense of word order. We fix this by adding a Positional Encoding vector to each word.

Notice how the same word gets a different final vector depending on its position.

02

Self-Attention Mechanism (Q, K, V)

This is the heart of the Transformer. It allows the model to look at other words in the sentence when processing a specific word. It uses Queries (Q) (what I'm looking for), Keys (K) (what I contain), and Values (V) (my actual content).

🎯 The "Bank" Example

Hover over a word to see which other words it pays "attention" to based on context.

💡 The Formula: Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V. The dot product of Query and Key creates an attention score, which dictates how much of the Value gets mixed in.
03

Multi-Head Attention

Instead of performing one single attention function, the model does it multiple times in parallel (multiple "heads"). One head might focus on grammar, another on pronouns, another on temporal relationships.

🧠 Parallel Thinking

Click different heads to see how they attend to different aspects of the sentence: "The ball hit the window and it broke."

04

Feed-Forward Network (FFN)

After attention mixes information between tokens, the FFN processes each token individually. This is where the model stores most of its "knowledge" and performs non-linear transformations using activation functions.

⚡ Activation Function Explorer

Modern LLMs like Llama use SwiGLU (a variant of Swish) for better gradient flow.

05

Layer Normalization

As data passes through dozens of layers, numbers can grow too large or shrink too small (exploding/vanishing gradients). LayerNorm (or RMSNorm) re-centers the values to have a mean of 0 and variance of 1.

⚖️ Vector Stabilization

Drag the slider to introduce extreme values. Watch LayerNorm instantly pull them back to a stable range.
