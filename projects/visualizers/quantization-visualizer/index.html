What is Quantization?

Quantization reduces the numerical precision of a model's weights. Instead of storing each weight as a 32-bit floating point number, we use fewer bits (16, 8, or even 4), dramatically reducing memory and speeding up inference.

🔢 Bit Precision Explorer

Drag the slider to see how the same number is represented at different bit widths. Watch how precision decreases as bits shrink.

Select a precision format to see how a number is stored in binary

💡 Key Insight: A 70B parameter model at FP32 needs 70B × 4 bytes = 280 GB of memory. At INT4, it only needs 70B × 0.5 bytes = 35 GB — an 8× reduction!

Weight Distribution

Neural network weights typically follow a bell curve (normal distribution) centered around zero. Quantization works well because most weights are small — only outliers suffer from clipping.

Quality vs Size Tradeoff

As you reduce precision, model quality (measured by perplexity) gradually degrades. The sweet spot depends on your use case — often INT4 with calibration retains 95%+ of quality at 8× compression.

📈 Perplexity vs Bit Width

Lower perplexity = better quality. Watch how quality degrades as bits decrease.

🎯 Sweet Spot: For most LLMs, INT4 (especially Q4_K_M in GGUF or AWQ) offers the best balance — ~95% quality at 8× compression. Going below 3 bits usually causes noticeable degradation.
