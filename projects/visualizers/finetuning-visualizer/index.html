02

Full Fine-Tuning

In Full Fine-Tuning, we update every single parameter in the model. If it's a 7 Billion parameter model, we are calculating gradients and updating 7 Billion weights.

📉 The Memory Wall

To train a 7B model, you need way more memory than just storing the model itself. You need space for Gradients and Optimizer States (Adam).

Model Weights (FP16)
14 GB
Gradients (FP16)
14 GB
Adam States (FP32)
56 GB
⚠️ The Problem: Full Fine-Tuning a 7B model requires ~84GB of VRAM. This won't fit on a single consumer GPU (like an RTX 4090 with 24GB). You'd need expensive server clusters.
03

LoRA (Low-Rank Adaptation)

Instead of changing the giant original weight matrix (W), LoRA freezes it. We then add two tiny, low-rank matrices (A and B) alongside it. We only train A and B.

🧩 Matrix Decomposition

W = W + (A × B). Watch how changing the "Rank" (r) affects the number of trainable parameters.

r=8
✅ The Magic: By training A and B instead of W, we reduce trainable parameters by 99%. When training is done, we simply multiply A×B and add the result back to W. There's zero inference delay!
04

QLoRA (Quantized LoRA)

LoRA fixes the gradient/optimizer memory, but the base model still takes up a lot of space. QLoRA goes a step further: it loads the base model in 4-bit precision (tiny!), but keeps the LoRA adapters in 16-bit.

⚡ Democratizing AI Training

Full Fine-Tuning (7B)
84 GB VRAM (Needs Multi-GPU)
Standard LoRA (7B)
30 GB VRAM (Needs 2x GPUs)
QLoRA (7B)
~10 GB VRAM (Fits on 1x RTX 3060)

When to Fine-Tune? (RAG vs FT)

A common mistake is trying to fine-tune a model to teach it new facts (e.g., "What is my company's refund policy?"). LLMs are bad at memorizing exact facts via training. Use RAG for facts, use Fine-tuning for style and behavior.
