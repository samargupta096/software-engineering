Domain 1: Agentic Architecture & Orchestration

Design multi-agent systems, manage task decomposition, and implement hub-and-spoke models.

The Agentic Loop

The core pattern: User Request → Send to Claude → Check stop_reason → Act → Loop.

stop_reasonMeaningAction
end_turnClaude finishedReturn response ✅
tool_useWants to call a toolExecute → return result → loop
max_tokensHit output limitSummarize/chunk (NOT "increase limit")
refusalSafety triggeredReport to user, don't retry
Exam Trap: "Increase max_tokens" is WRONG. It controls output length, not context. Use /compact or subagents.

4 Multi-Agent Patterns

PatternWhenKey Principle
Hub-and-spokeMultiple independent subtasksCoordinator delegates, subagents return summaries
Evaluator-optimizerQuality mattersGenerate → evaluate → loop until quality
PipelineSequential dependenciesOutput A = Input B
Parallel fan-outIndependent researchSpawn N subagents simultaneously

5 Golden Rules

  • Context isolation — each subagent has its OWN context window
  • Pass summaries, not histories — prevents context pollution
  • Manifest files for crash recovery
  • Subagents cannot share memory — communicate only through coordinator
  • Limit tools: 4-5 per agent — more = more selection ambiguity

Domain 3: Prompt Engineering & Structured Output

Structured Output via Forced Tool Use

1. Define a "tool" with your JSON Schema
2. Set tool_choice: {"type":"tool","name":"extract_X"}
3. Claude MUST return data matching the schema

tool_choiceBehavior
"auto"Claude decides (default)
"any"Must use some tool
{"type":"tool","name":"X"}Forces specific tool
"none"No tools allowed
Exam Trap: "Silently fix Claude's output errors" is WRONG. Always send validation errors back to Claude.

Anti-Hallucination Patterns

PatternWhy
"type":["string","null"]Claude returns null instead of guessing
"enum":[...,"other"] + detailHandles unexpected values
Required vs optional fieldsExplicit about what must exist

Active Recall Flashcards

Click to flip. 16 high-yield cards covering the most tested concepts.

What are the 4 stop_reason values?

end_turn (done), tool_use (call tool), max_tokens (output limit hit), refusal (safety block)

Minimum tokens for Prompt Caching?

1024 tokens. Any prefix shorter than this won't be cached.

How to run Claude Code in CI/CD?

Use the -p (print) flag for non-interactive mode. Add --output-format json for structured output.

Grep vs Glob?

Grep = search inside file contents. Glob = search file names/paths by pattern.

MCP: Tools vs Resources vs Prompts?

Tools = actions with side effects. Resources = read-only data. Prompts = reusable templates.

What is the Lost-in-the-Middle effect?

LLMs have U-shaped attention. They attend most to content at the beginning and end, least in the middle.

How many tools should each agent have?

4-5 tools max. More tools = more selection ambiguity = more errors.

What does fork_session do?

Creates a branch from the current session for safe exploration without polluting the main session.

How to guarantee strict JSON output?

Use Tool Use API with tool_choice: {"type":"tool","name":"X"}. Never rely on prompt instructions alone.

What is custom_id in Batch API?

A unique identifier per batch request for result correlation. Results are NOT returned in submission order.

PreToolUse vs PostToolUse hooks?

PreToolUse = validate/block before execution. PostToolUse = act after completion (e.g., auto-format after Write).

What should subagents return to coordinators?

Structured summaries with key findings, NOT full document contents. Full content = context pollution.

3 types of tool errors?

Transient → retry. Business logic → return to Claude. Permission → escalate, never retry.

Where does path-scoped config go?

.claude/rules/*.md with YAML frontmatter globs: patterns. Auto-loads when editing matching files.

Message Batches API cost savings?

50% cheaper than real-time. Max 100K requests or 256MB. Up to 24h SLA. No multi-turn tool calling.

When to use Plan Mode vs Direct?

Plan Mode: multi-file, architectural, high-risk, ambiguous. Direct: single file, obvious bug fixes, low-risk.
